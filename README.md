# STECH — Southwest Technical College WordPress theme

Custom, block-based WordPress theme for Southwest Technical College (STECH),
replacing the legacy Divi build. Composed entirely of **Gutenberg / ACF blocks**
against the approved STECH brand system. Depends only on **ACF PRO**,
**Gravity Forms**, and **Yoast SEO** (plus The Events Calendar and WP Job Manager
for the events/jobs features that are being carried over).

Built by [Quick Brown Fox](https://quickbrownfox.co).

---

## Architecture

Single standalone theme (no parent). Concerns are split into small modules.

```
stech/
├── functions.php            Thin loader — requires inc/* in order
├── header.php / footer.php  Global chrome (menus + Theme Settings options)
├── page.php / front-page.php Block-composed pages (the_content)
├── singular.php index.php archive.php search.php 404.php
├── inc/
│   ├── setup.php            Theme supports, nav menus, image sizes
│   ├── enqueue.php          Fonts + compiled CSS/JS (front-end & editor)
│   ├── post-types.php       CPTs: Program, Staff (+ taxonomies); Posts→"News"
│   ├── blocks.php           block.json auto-loader, STECH category, allowed types
│   ├── acf.php              ACF options pages, JSON sync, dependency notices
│   └── helpers.php          stech_svg(), stech_link(), stech_nav() walker, breadcrumb
├── blocks/<slug>/           One folder per ACF block
│   ├── block.json           name: "stech/<slug>", acf.renderTemplate
│   └── render.php           Markup; reads fields via get_field()
├── acf-json/                ACF local JSON (field groups, version-controlled)
├── src/
│   ├── sass/                app.scss → base/ (tokens, components) + blocks/
│   ├── js/                  main.js → blocks.js + partials/
│   └── img/shared/          Brand SVGs (logo, paw, Ursa) — NOT photographic media
└── public/                  Compiled app.css / app.min.js (committed, served)
```

### The brand system is the styling source of truth

`src/sass/base/_tokens.scss`, `_components.scss`, and `_components-extended.scss`
are the **approved** STECH CSS, verbatim. They already contain the styles for all
46 blocks plus the WCAG 2.2 work (`:focus-visible`, reduced-motion, ARIA-state
styling). **Rule: no raw hex/px/rem that isn't a token.** A block's `render.php`
only needs to output markup with the same class names — styling comes for free.

## Building a block

1. `blocks/<slug>/block.json` — `"name": "stech/<slug>"`, `"acf": { "renderTemplate": "render.php" }`.
2. `blocks/<slug>/render.php` — reads `get_field('<slug>_*')`, outputs the approved markup. Wrap the root element with `stech_block_attrs( $block, '<base-class>' )`.
3. `acf-json/group_stech_<slug>.json` — field group with `location` `block == stech/<slug>`; field names prefixed with the block slug.

The auto-loader (`inc/blocks.php`) registers every `blocks/*/block.json` on `init`.

## Build

The compiled assets in `public/` are committed, so the theme runs without a build
step. To rebuild after editing `src/`:

```bash
npm install
npm run build      # dart-sass + esbuild → public/css/app.css, public/js/app.min.js
npm run watch      # dev rebuild on change
```

> Note: `public/` may also be regenerated by concatenating `src/sass/base/*` (the
> brand CSS is plain CSS) if the node toolchain is unavailable.
